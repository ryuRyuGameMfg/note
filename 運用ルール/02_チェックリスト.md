## Markdown安定運用ルール

### 引用ブロック（>）の注意点

**❗ 引用内は必ずプレーンテキストのみ**

禁止パターン
```
> **太字**  ← NG
> 1. 数字箇条書き  ← NG
> - ハイフン箇条書き  ← NG
```

推奨パターン
```
> テキストのみ
> ❶ 囲み数字で代替
> ・中点で代替
```

### `> ▼` の直後に置いてはいけない要素
- `-` ハイフン箇条書き
- `1.` 数字箇条書き
- ` ``` ` コードブロック
- `##` 見出し

### 回避策

パターンA：引用を閉じる
```
> ▼ 注意点

・項目1
・項目2
```

パターンB：ワンクッション入れる
```
> ▼ 注意点
> ｜詳細
> ・項目1
> ・項目2
```

### 引用とコードブロックの隣接
- 引用直後にコードブロックを置かない
- 間に通常文1行を挟む
