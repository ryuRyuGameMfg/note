# 執筆ガイド

## 記事ライフサイクル

```
アイデア → 執筆中 → 公開準備完了 → 公開 → 分析 → 改善
```

### 原則
- 1記事につき1ファイルのみ保持
- 次の段階に進む際は、古い段階のファイルを削除して移動

### フォルダ構成

**note記事**
```
note/
├── 01_アイデア/
├── 02_下書き/
│   └── 02_執筆中/
├── 03_公開準備完了/
├── 04_公開済み/
└── 05_分析/
```

**X投稿**
```
X/
├── 01_アイデア/
├── 02_下書き/
├── 03_公開準備完了/
├── 04_公開済み/
└── 05_分析/
```

## note記事の書き方

### タイトルの黄金則

**数字 + 具体的な成果 + ターゲット明確化**

✓ 効果的な要素
- 具体的な数字（期間、金額、倍率）
- 明確な成果やベネフィット
- 想定読者が自分事と感じる表現

✗ 避けるべきパターン
- 抽象的で曖昧な表現
- 読者メリットが不明確
- ターゲットが広すぎる

### 冒頭で心を掴む（最重要）

**最初の3行で読者の共感を得る**

効果的な冒頭パターン
- 読者の悩みや課題の言語化
- 共感を呼ぶ体験談
- 意外性のある事実や数字
- 問いかけによる巻き込み

### 基本構成（2500-4000文字）

1. フック（200字）- 共感できる課題提示
2. 問題提起（300字）- なぜ重要か
3. 解決策（1500字）- 具体的な方法
4. 実践結果（800字）- Before/After
5. まとめ（200字）- 行動喚起

### ストーリー型記事の構成

```
フック（衝撃的な一文）
↓
過去の状況（共感を得る）
↓
きっかけ（変化の始まり）
↓
失敗・葛藤（リアリティ）
↓
転機（発見・気づき）
↓
成功体験（具体的な効果）
↓
デメリットも正直に（信頼性）
↓
読者へのメッセージ（行動喚起）
```

### 人間味を出す文体

✗ 避けるべき表現
- 「以下の通りです」
- 「〜することができます」
- 「効果的です」

✓ 代わりに使う表現
- 「こんな感じ」
- 「〜できる」
- 「めっちゃいい」

## noteの装飾ルール

### 見出し
- 2階層まで（## と ### のみ）
- 【第○章】形式でストーリーの流れを明確化

### 太字
- 1段落に1〜2個まで
- 製品スペック、感情的な強調、インパクトある表現に使用

### 引用ブロック（>）
**❗ 重要：引用ブロック内は装飾なしのプレーンテキストのみ**

用途
- 内心の声・つぶやき
- 重要なメッセージ
- 悩みの可視化

禁止事項
- 引用内に太字（**）や斜体（*）を使用しない
- 引用内に数字箇条書き（1. 2. 3.）を使用しない
- 空の引用行（> のみ）を作らない

### リスト表現

番号付きリスト（囲み数字を使用）
```
❶ 項目1
❷ 項目2
❸ 項目3
```

ポジティブ要素
```
✓ メリット1
✓ メリット2
```

ネガティブ要素
```
▼ デメリット1
▼ デメリット2
```

### URL配置

**必ず独立した行に配置**
```
詳細はこちら：
https://example.com/
```

### 水平線（---）

原則として使用しない（noteで表示を阻害する可能性あり）

## CTA配置戦略

### 配置位置
1. 導入直後（本編開始前）
2. 記事末尾（まとめの後）

### 統一フォーマット
```
**ゲーム開発のご相談：**
https://ryuryugame.netlify.app/

**AI×サイネージ×アバター接客ならお任せ：**
https://guidecastai.netlify.app/
```

### 注意点
- CTAの後に追伸は書かない
- 押し付けがましくしない
- 記事内容と関連性を保つ

## X投稿の書き方

### 基本構成
```
フック（1行目で目を引く）
↓
本文（具体的な内容・価値）
↓
CTA（リンク or 行動喚起）
```

### 効果的なパターン

**課題解決型**
```
「〇〇できない」
「△△したい」
「でも□□」

そんな悩み、解決します

具体的なメリット
URL
```

**実績訴求型**
```
【実績数字】
何をしたか

できること：
・項目1
・項目2
・項目3

URL
```

### 投稿時間
- 土曜日 10:00-12:00
- 平日夜 20:00-22:00

## 品質ゲート（公開基準）

### 必須条件
- 構成と文字数：基本構造かつ2500-4000字
- 視覚最適化：引用ブロック優先、URL独立行
- 権威性・実績：数字で根拠を提示
- デメリットの明示：注意点・落とし穴を正直に記載
- CTA：導入直後と末尾の2箇所

### コピペ安定性
- 引用内の数字箇条書き回避
- 空行「>」なし
- 水平線は極力回避
