# X運用方針

最終更新: 2025年12月

## コンセプト

ミッション: ゲーム開発のリアルを、カジュアルに発信する

ポジショニング:
- 誰が: 235件実績の24歳個人開発者（りゅうや）
- 何を: 開発の日常、失敗、学び、進捗
- 誰に: ゲーム開発者、個人開発者、副業に興味ある人
- どう: 友達に話すようなカジュアルな口調で

noteとの役割分担:

| プラットフォーム | 役割 | 内容 |
|-----------------|------|------|
| X | 日常・関係構築 | 進捗、気づき、失敗談、note告知 |
| note | 詳細・信頼構築 | 体系的なノウハウ、実績、深掘り |

導線: X（認知・共感）→ note（信頼・詳細）→ ココナラ（行動）

## ターゲット

メインターゲット:

| 項目 | 詳細 |
|------|------|
| 属性 | 20-35歳、男性中心、エンジニア or 開発に興味ある層 |
| 関心 | Unity、ゲーム開発、個人開発、副業、AI |
| 行動 | 技術情報を収集、同じ開発者をフォロー |

フォロワー層:
- ゲーム開発者（Unity/Unreal）
- 個人開発者・フリーランス
- プログラミング初心者〜中級者
- 副業・複業に興味がある人
- AI技術に興味がある人

## KPI・目標

短期目標（3ヶ月）:

| 指標 | 目標値 |
|------|--------|
| フォロワー | 1,000 |
| 平均インプレッション | 1,000/投稿 |
| 平均エンゲージメント率 | 3% |
| note記事告知のクリック | 50/回 |

中期目標（1年）:

| 指標 | 目標値 |
|------|--------|
| フォロワー | 5,000 |
| 平均インプレッション | 5,000/投稿 |
| note流入 | 月500 |

重要指標の優先順位:
1. note/ココナラへの流入（収益直結）
2. 返信数（アルゴリズム的に27倍の価値）
3. エンゲージメント率（投稿の質）
4. フォロワー数（リーチの基盤）

## コンテンツ戦略

投稿カテゴリと比率:

| カテゴリ | 割合 | 目的 |
|---------|------|------|
| 開発進捗・日常 | 40% | 親近感、継続性 |
| 失敗談・学び | 25% | 共感、信頼 |
| Tips・気づき | 20% | 価値提供、保存 |
| note/サービス告知 | 15% | 流入、収益 |

鉄則: 宣伝2：日常8
- 宣伝ばかり → フォロー解除される
- 日常だけ → 収益に繋がらない

投稿の役割:
1. 日常投稿: 親近感を持ってもらう
2. 価値投稿: この人の発信は役立つ
3. 告知投稿: noteを読んでみよう
4. note記事: 信頼を深める
5. CTA: ココナラで相談

## アルゴリズム対応（公式情報）

エンゲージメント重み付け:

| アクション | 重み（いいね=1） | 対策 |
|-----------|-----------------|------|
| いいね | 1 | - |
| リツイート | 2 | 保存したくなる内容 |
| 返信 | 27 | 質問で締める |
| 返信への返信 | 150 | 必ず返信する |

必須対応:

| 項目 | 対応 |
|------|------|
| 質問で締める | 「皆さんはどうしてますか？」 |
| 返信には必ず返信 | 会話を続ける |
| 外部リンクは返信欄 | 本文にURLを入れない |
| 画像・動画添付 | 1.7倍拡散効果 |

やってはいけないこと:
- 本文に外部リンクを入れる
- URLのみの投稿
- 宣伝ばかりの投稿
- 返信を無視する
- 大量フォロー/アンフォロー

## トーン&マナー

基本トーン: カジュアルで親しみやすい、友達に話すような口調

使う表現:
- 「〜なんですよね」「〜なんです」
- 「めちゃくちゃ〜」「マジで〜」
- 「やったー！」「おかげさまで」
- 「ぜひ〜してみて」

避ける表現:
- 「〜いたします」「〜でございます」（硬すぎ）
- 「ご検討ください」（営業っぽい）
- 「是非ともご利用を」（宣伝臭い）

一人称:
- 「僕」「俺」（カジュアルな場面）
- 「私」（フォーマルな場面）

## 運用スケジュール

投稿頻度:

| 項目 | 目標 |
|------|------|
| 日常投稿 | 週2-3回 |
| note告知 | 記事公開時 |
| 返信対応 | 毎日確認 |

投稿時間:

| 時間帯 | 優先度 | 理由 |
|--------|--------|------|
| 平日 20:00-22:00 | 高 | 帰宅後のゴールデンタイム |
| 土曜 10:00-12:00 | 中 | 朝のゆっくり時間 |
| 平日 12:00-13:00 | 中 | 昼休み |

週間ルーティン:
- 月: 開発進捗 or 週末の振り返り
- 火-木: 日常、気づき、Tips（1-2回）
- 金: 来週の予告 or 振り返り
- 土: note記事告知（公開日）
- 日: 休み or 軽い日常投稿

## 投稿パターン

パターン1: 開発進捗（日常）
```
今日の開発メモ。

〇〇の機能、ようやく動いた！
△△で苦戦したけど、□□で解決。

皆さんは〇〇で困った経験ありますか？

#Unity #ゲーム開発
```

パターン2: 失敗談（共感）
```
正直に言う。

最初の見積もり、甘すぎた。
時給換算したらバイト以下だった...

今は〇〇を意識してる。

同じ経験ある人いますか？

#個人開発 #フリーランス
```

パターン3: Tips（価値提供）
```
個人開発で大事だと思うこと。

・〇〇する
・△△を意識
・□□を避ける

これだけで235件やってこれた。

皆さんが大事にしてることは？

#個人開発 #Unity
```

パターン4: note告知（リンクは返信欄）
```
【新記事】〇〇について書きました！

・△△の方法
・□□のコツ
・実際の結果（数字あり）

読んだ感想聞かせてください！
詳細はリプ欄に

#note #ゲーム開発
```

## 改善サイクル

週次確認:
- インプレッション/エンゲージメント確認
- 伸びた投稿の共通点分析
- 返信への対応漏れチェック

月次振り返り:
- フォロワー増減
- 最も伸びた投稿TOP3
- note/ココナラへの流入数
- 宣伝と日常の比率チェック

成功パターンの蓄積:
- 1行目のフック
- 投稿時間
- 画像の有無
- 質問の仕方

## 注意事項

やること:
- 返信には必ず返信する（150倍の価値）
- 質問で締める（27倍の価値）
- リンクは返信欄に配置
- 画像・動画を積極活用
- 宣伝2：日常8を守る

やらないこと:
- 本文に外部リンクを入れる
- 宣伝ばかり投稿
- 返信を無視
- 炎上しそうな発言
- 攻撃的・上から目線
