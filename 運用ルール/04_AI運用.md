# AI運用ガイド

## 基本原則

### 単一参照ファイル原則
- セッション開始時に参照するナレッジを1ファイルに限定
- 既定：`運用ルール/01_執筆ガイド.md`

### 計画駆動
- 実装前に「計画書」を生成し、合意後に実行
- 変更理由の明示：ナレッジ更新時は「変更理由/根拠/反映箇所」を必ず記載

## 推奨フロー（1セッション）

### 1. インテイク（要件整形）
- 目標/制約/入出力/評価を短文化
- 参照する単一ナレッジファイルを宣言

### 2. 計画（Plan）
- 計画書テンプレートに沿って作成
- 小タスクは3手順以内に収める

### 3. 合意
- 原則は承認後に実行
- 非対話時は「承認省略モード」で続行

### 4. 実行（Do）
- 計画の手順に沿って実装
- 作業単位ごとに簡易検証→通過後に次へ

### 5. 検証（Check）
- チェックリストで品質/整合性を確認
- 失敗時は計画へロールバック

### 6. 反映（Act）
- ナレッジ更新が必要なら「変更理由/差分/根拠」を併記
- 分析ログを保存（任意）

## プロンプト集

### セッション開始プロンプト

```
セッション開始。参照: 運用ルール/01_執筆ガイド.md

与件
- 目的: {達成したいこと}
- 制約: {時間/範囲/禁止事項}
- 入出力: {入力ソース/欲しい最終出力}
- 評価基準: {合格条件/KPI}
- 承認モード: {承認必須 | 承認省略}

手順
1) 必要資料を最大3点選定（相対パスと選定理由を列挙）
2) 計画書テンプレートに準拠したPLANを作成
3) PLAN承認後、手順ごとに 実行→検証→合否→次へ を実施
4) 実行中は選定資料のみ参照。note特有ルールを厳守
5) 完了時、チェックリストで最終確認
6) 必要ならナレッジ更新案を提示

出力
- 選定資料（相対パス/理由）
- PLAN（計画書テンプレート準拠）
- 実行ログ（変更点/理由/検証結果）
- 成果物（修正文や追加ファイル）
- 最終チェック結果とナレッジ更新案
```

### 計画作成プロンプト

```
以下の与件から、単一参照ファイル原則に従ってPLANを作成してください。

与件
- ゴール: {目標}
- 制約: {制約条件}
- 入出力: {入力/出力}
- 評価観点: {評価基準}

要件
1) 目的と評価基準を明確化
2) 参照ナレッジを1つだけ宣言
3) 成果物/手順/リスク/検証/反映/承認を具体化
4) 小タスクは3手順以内に収める

出力は計画書の章立てのみで提出。
```

### 実行プロンプト

```
承認済みPLANに沿って実行してください。

要件
- 手順ごとに「実行→検証→合否→次へ」の順で進める
- 変更は小さく分割し、各手順のDoDを満たしたら次へ
- 失敗時はPLANの該当手順を最小修正して再実行案を提示
- 反映が必要なナレッジがあれば、変更理由/反映箇所/根拠を出力末尾に提案

出力
- 実行ログ: 何をどこにどう変えたか（理由付き）
- 成果物: 修正済みファイル
- 検証結果と残課題
```

### 骨子生成プロンプト

```
「{記事タイプ}」の記事骨子を作成。

ペルソナ: {ターゲット/悩み}
使える実績: {実績・数値}

要件
- 基本構造（フック/問題/解決/結果/まとめ）
- 見出しは2階層まで
- デメリットも明記
- CTAは導入直後と末尾に配置

出力
- 見出し（##, ###まで）
- 各節の要点（3–5点）
- CTA配置（導入直後/末尾）
```

### SEO/AEO最適化プロンプト

```
以下の要件で本文を最適化してください。

メインKW: {キーワード}
サブKW: {サブキーワード}

要件
1) note: 見出しは##/###、引用内装飾禁止、`> ▼` 直後に `-`/`1.`/```/## を置かない
2) Google: E-E-A-T（経験・実績・出典）明示、検索意図網羅、モバイル可読
3) AEO: FAQ 3問（質問→1–3文回答）、冒頭要約100–200字、手順は箇条書き短文
4) タイトルは「数字+成果+ターゲット」

出力
- 改善提案（タイトル/導入/見出し/本文/FAQ/CTA/内部リンク）
- 修正本文（note仕様厳守）
```

## ナレッジ更新の基準

更新すべき場合
- 再現性のある不具合回避策が確認できた
- 運用ルールの明確化・簡素化で誤用が減る
- 既存ルールと矛盾しない/もしくは上位互換に置換できる
- 出典/一次情報に基づく（推測は明記）

## ファイル運用

### 計画書
- 保存先：`note/05_分析/セッション計画/`
- 命名：`yyyymmdd_hhmm_タイトル.md`

### セッションログ
- 保存先：`note/05_分析/セッションログ/`
- 命名：`yyyymmdd_hhmm_タイトル.md`

### 変更原則
- 小さく・明確に
- 複数目的の混在を避ける

## 役割分担

### 人（あなた）
- 目標/優先度/判断/承認

### AI
- 計画作成・実行・検証・ナレッジ反映案の提示
