# AI運用ガイド

AIツール（Claude、ChatGPT等）を活用した記事作成・投稿作成のルール。

まず 00_はじめに.md を読むこと。読む順番が書いてある。

## 読む順番

note記事作成:
1. 01_運用方針/note運用方針.md
2. 04_SEO/公式アルゴリズム情報.md
3. 02_note/テンプレート.md
4. 02_note/チェックリスト.md
5. 出力

X投稿作成:
1. 01_運用方針/X運用方針.md
2. 04_SEO/公式アルゴリズム情報.md
3. 03_X/テンプレート.md
4. 03_X/チェックリスト.md
5. 出力

## セッション構成

1. 計画（PLAN）: 目的・成功条件の明確化、参照ナレッジの指定
2. 実行（DO）: 手順単位で実行→検証、小分けで確実に
3. 検証（CHECK）: チェックリスト通過確認、品質ゲート確認
4. 改善（ACT）: ナレッジ更新、次回への反映

## note記事作成時

計画書テンプレート:
```
目的:
成功条件/KPI:

参照ナレッジ:
- 01_運用方針/note運用方針.md
- 04_SEO/公式アルゴリズム情報.md
- 02_note/テンプレート.md

アルゴリズム対応チェック:
- [ ] 冒頭3行に要約（自己紹介・広告NG）
- [ ] ハッシュタグ3〜5個
- [ ] 公開後すぐにXで告知予定

検証: 02_note/チェックリスト.md
```

必須確認事項:

| 項目 | 確認内容 |
|------|---------|
| 冒頭3行 | 要約になっているか（自己紹介NG） |
| ハッシュタグ | 3〜5個か |
| CTA | 2箇所（導入後・末尾）あるか |
| 文字数 | 2500-4000字か |

## X投稿作成時

計画書テンプレート:
```
目的:
成功条件/KPI:

参照ナレッジ:
- 01_運用方針/X運用方針.md
- 04_SEO/公式アルゴリズム情報.md
- 03_X/テンプレート.md

アルゴリズム対応チェック:
- [ ] 質問で締める（返信獲得）
- [ ] 外部リンクは返信欄
- [ ] 画像添付予定

検証: 03_X/チェックリスト.md
```

必須確認事項:

| 項目 | 確認内容 |
|------|---------|
| 質問で締める | 返信を促しているか |
| 外部リンク | 本文に入れていないか |
| ハッシュタグ | 2-4個か |

## セッション実行チェック

事前:
- [ ] 00_はじめに.md を確認
- [ ] 読む順番に従って参照
- [ ] PLAN作成・承認

実行中:
- [ ] 手順単位で実行→検証→合否判定
- [ ] 変更は小分け、DoDを満たしてから次へ
- [ ] 想定外はPLANを最小修正

事後:
- [ ] チェックリスト通過
- [ ] ナレッジ更新（変更理由/根拠/反映箇所）

## よくある指示例

note記事作成:
```
note記事を作成してください。
テーマ：〇〇
ターゲット：△△
参照順：01_運用方針/note運用方針.md → 04_SEO/公式アルゴリズム情報.md → 02_note/テンプレート.md → 02_note/チェックリスト.md
```

X投稿作成:
```
X投稿を作成してください。
内容：〇〇
参照順：01_運用方針/X運用方針.md → 04_SEO/公式アルゴリズム情報.md → 03_X/テンプレート.md → 03_X/チェックリスト.md
```

note記事告知（X投稿）:
```
note記事の告知投稿を作成してください。
記事タイトル：〇〇
記事URL：https://note.com/xxx/n/xxx
参照：04_SEO/公式アルゴリズム情報.md、03_X/テンプレート.md（note記事告知型）
注意：URLは返信欄に配置、本文には「リプ欄に」と明記、質問で締める
```

## ナレッジ更新ルール

更新タイミング:
- 新しい公式情報が出たとき
- 運用で気づきがあったとき
- 失敗パターンを発見したとき

更新方法:
1. 変更理由を明確化
2. 変更箇所を特定
3. 最小限の修正
4. 関連ドキュメントの参照更新
