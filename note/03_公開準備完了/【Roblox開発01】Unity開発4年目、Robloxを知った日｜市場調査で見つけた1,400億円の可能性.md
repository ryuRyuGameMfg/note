## Robloxって、そもそも何？

**Roblox（ロブロックス）**は、ゲーム開発と収益化ができるプラットフォーム。

**基本スペック：**

## どうやって作るのか？Roblox Studioと開発言語

### Roblox Studioで開発

**Unity開発者**なら、すぐ理解できる。

**Roblox Studio = Unity Editorみたいなもの**

**できること：**
- 3D空間の設計・構築
- オブジェクトの配置
- スクリプトで動作を実装
- マルチプレイヤー機能（標準実装）
- 公開して収益化

**Unity開発者**として驚いたのは、**マルチプレイヤーが標準実装**されてること。

**Roblox Create（公式開発ポータル）：**
https://create.roblox.com/

### 使用言語：Luau（Luaベース）

**Luau（ルアウ）**という言語を使う。

**特徴：**
- ベース：Lua 5.1
- Roblox向けに最適化・拡張
- PythonやJavaScriptに似てる

**実際のコード例：**

```lua
-- プレイヤーが参加したら
game.Players.PlayerAdded:Connect(function(player)
    print(player.Name .. "が参加しました")
end)
```

**Unity開発者**として感じたこと：
- イベント処理はC#のデリゲートに似てる
- 構文がシンプルで読みやすい
- セミコロン不要

**エンジニア**として**副業・複業**で**Roblox開発**を始めるなら、言語の習得は1〜2日で可能。

## Unity vs Roblox：決定的な3つの違い

**Unity開発者**として感じた、決定的な違い。

### 違い①：集客が自動

**Unityの場合：**
- Steam、App Storeでリリース
- SEO、SNS、広告で集客
- マーケティングコストが高い

**Robloxの場合：**
- Robloxプラットフォーム内でリリース
- 1億1,180万人のユーザーに自動リーチ
- マーケティングコストがほぼゼロ

**エンジニア**として**副業・複業**で稼ぐなら、この集客力の差は圧倒的だと思った。

### 違い②：サーバー管理不要

**Unityの場合：**
- 自前でサーバー構築
- Photon、Mirror等のライブラリ
- 開発期間：数週間〜数ヶ月

**Robloxの場合：**
- マルチプレイヤーが標準実装
- サーバー管理は自動
- 開発期間：ゼロ

**Unity開発者**として、この差に驚いた。

### 違い③：言語がシンプル

**Unityの場合：**
- C#（型定義、セミコロン必須）
- やや複雑

**Robloxの場合：**
- Luau（動的型付け、セミコロン不要）
- シンプルで学習しやすい

**エンジニア**として**副業・複業**で**Roblox開発**を始めるなら、この学習コストの低さは魅力的。
