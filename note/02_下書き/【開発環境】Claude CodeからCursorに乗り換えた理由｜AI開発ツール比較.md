## Claude Codeとは

Claude Codeは、Anthropic社が提供するAIコーディングツールです。Claude（クロード）というAIモデルを使って、ターミナル上でコードを書いたり、ファイルを編集したりできます。

Anthropic公式：
https://www.anthropic.com/claude

### 良かった点

- Claude Opusの性能が高い
- 複雑なタスクも理解してくれる
- コードの品質が良い
- 長文の理解力が優れている

特に、長い仕様書を読み込ませて「この通りに実装して」みたいな使い方が得意でした。

### 困った点

- 制限がすぐ来る
- UIがターミナルベースで見にくい
- 出力が流れていくので、後から見返しにくい
- コピペが面倒

使用制限に引っかかると、しばらく待たなきゃいけないのがストレスでした。

## Cursorとは

CursorはVS Codeベースのエディタで、AIコーディング機能が統合されています。Claude、GPT-4、Geminiなど複数のモデルを使い分けられるのが特徴。

Cursor公式：
https://cursor.com/

## 乗り換えた5つの理由

### 理由1：UIがめちゃくちゃ見やすい

これが**1番大きい**かもしれない。

Claude Codeはターミナルベースなので、出力がずらーっと流れていく。

問題点：
- スクロールして過去の出力を探すのが大変
- コードブロックの区切りが分かりにくい
- マークダウンのレンダリングがない

Cursorはチャット形式で、コードブロックやマークダウンでメリハリがある。

しかも、コードブロックで出力されたコードを**ワンクリックでコピペできる**。

Claude Codeだと、範囲選択してコピーする必要がありました。

### 理由2：作業のまとめが見やすい

Cursorだと、作業が終わった後にまとめを出してもらえます。

- 表形式で変更点を整理
- 図式で構造を説明
- 差分を分かりやすく表示

プロンプトで「最後にまとめて」って言えば、きれいに整理してくれる。

Claude Codeだと、ターミナルの出力をスクロールして確認する必要がありました。

### 理由3：色んなモデルが使える

Claude Codeだと、Claudeしか使えない。

Cursorは**Claude、GPT-4、Gemini、o1など複数のモデルが使える**。

メリット：
- 制限が来ても別のモデルに切り替えられる
- タスクに応じて最適なモデルを選べる
- 無料モデルも選択可能

どうせ制限が来るなら、色んなモデル使えた方がいいですよね。

### 理由4：並列作業に対応

複数のエージェントを立ち上げて、同時に作業できます。

- フロントエンドの修正をしながら
- 別のエージェントでバックエンドを書く
- 並行してドキュメントを作成

Claude Codeも対応してましたが、UIが見やすいCursorの方が使いやすいです。

### 理由5：VS Codeの拡張機能が使える

Cursorは VS Codeベースなので、既存の拡張機能がそのまま使えます。

僕が使ってる拡張機能：
- Prettier：コード整形
- ESLint：コード品質チェック
- GitLens：Git履歴の可視化
- Unity Code Snippets：Unity開発支援

VS Codeから乗り換える場合、設定もそのまま引き継げます。

## モデルの使い分け

### Opus（高性能モデル）

使い所：
- 複雑なタスク
- 設計の相談
- 大規模なリファクタリング
- 新しい技術の学習

高コストだけど、精度が高い。

### Sonnet（標準モデル）

使い所：
- 日常的なコーディング
- 小〜中規模の修正
- ドキュメント作成
- デバッグ支援

Sonnetでも十分優秀なので、普段はSonnetで回して、難しいタスクだけOpusを使う感じ。

### o1（推論特化モデル）

使い所：
- 論理的な思考が必要なタスク
- アルゴリズム設計
- 複雑なバグの原因究明

最近追加されたモデルで、推論能力が高いです。

## まとめ

Claude CodeからCursorに乗り換えた理由：

1. UIがめちゃくちゃ見やすい（1番大きい）
2. コードブロックのコピペが簡単
3. 作業のまとめが分かりやすい
4. 色んなモデルが使える
5. VS Code拡張機能が使える

結論、**Cursorに乗り換えて良かった**です。

特にUIの見やすさとコードブロックのコピペは、Claude Codeにはなかった強み。
