## Cursor Ultraプランの詳細（2025年12月最新）

### 料金と含まれる使用量

**月額料金：$200/月**

**含まれる使用量：**
- **$400相当のAPI agent usage**
- **約4,500 requests/月**（Claude Sonnet 4使用時）
- **約11,000 requests/月**（Gemini使用時）
- **約10,000 requests/月**（GPT-5使用時）

**公式ドキュメント：**
https://docs.cursor.com/account/rate-limits

### 使用量のリセットタイミング

**重要ポイント：日次制限ではなく月次制限**

Cursor Ultraプランは**月次でリセット**されます。
サブスクリプション開始日に基づいて、毎月同じ日にリセットされます。

例：
- 12月17日に契約開始 → 毎月17日にリセット
- 1月23日に契約開始 → 毎月23日にリセット

**つまり：**
- 1日目に150リクエスト使っても、2日目も150リクエスト使える
- 月間の合計が4,500リクエスト以内であれば問題なし
- 月の前半に集中して使っても、後半も使える

これは**柔軟性が高い**点として評価できます。

### 超過時の対応

月間の使用量を超えた場合：

1. **エディタ内で通知**
2. **追加利用の選択肢：**
   - 追加使用量を購入（API料金+20%マークアップ）
   - より上位のプランにアップグレード

**追加料金の例（Claude Sonnet 4.5使用時）：**
- 入力トークン：$3.60/100万トークン
- 出力トークン：$18/100万トークン

**公式ドキュメント：**
https://docs.cursor.com/account/rate-limits

### レート制限について

**レート制限は数時間ごとにリセット**されます。
これはシステムの安定性を保つための仕組みで、
連続して大量のリクエストを送る場合に影響します。

ただし、通常の開発作業では問題になることはほとんどありません。

## Claude Code Max 20xプランの詳細（2025年12月最新）

### 料金と含まれる使用量

**月額料金：$200/月**

**含まれる使用量：**
- **約900メッセージ/5時間**
- **約4,500メッセージ/日**（理論値）
- Proプランの20倍の使用量

**公式情報：**
https://www.claude.com/pricing

### 使用量のリセットタイミング

**重要ポイント：5時間ごとの制限**

Claude Code Max 20xプランは**5時間ごとにリセット**されます。

**つまり：**
- 5時間以内に900メッセージまで使える
- 5時間経過すると、また900メッセージ使える
- 理論上は1日で約4,500メッセージ使える計算

ただし、**2025年8月から週次レート制限も追加**されました。
これは全ユーザーの5%未満に影響する制限で、
過度な使用を防ぐためのものです。

**公式情報：**
https://support.anthropic.com/en/articles/11014257-about-claude-s-max-plan-usage

### 超過時の対応

5時間の制限に達した場合：

1. **次の5時間ウィンドウまで待機が必要**
2. **追加購入オプションなし**（制限内での使用のみ）

ただし、**「extra usage」機能**で追加利用は可能です。
この場合、標準のAPI料金が適用されます。

**API料金の例（Claude Sonnet 4.5使用時）：**
- 入力トークン：$3/100万トークン
- 出力トークン：$15/100万トークン

**公式情報：**
https://support.claude.com/en/articles/12429409-extra-usage-for-paid-claude-plans

## コストパフォーマンス徹底比較

### 1日の使用量比較

| 項目 | Cursor Ultra | Claude Code Max 20x |
|------|-------------|---------------------|
| **1日の理論的使用量** | 約150 requests/日<br>（4,500÷30日） | 約900メッセージ/5時間<br>（理論上は4,500/日） |
| **制限の種類** | 月次制限 | 5時間ごとの制限 |
| **柔軟性** | 高い（月内で自由に配分） | 中（5時間ごとにリセット） |

### コストパフォーマンスの計算

**Cursor Ultraの場合：**
- 月額$200で約4,500 requests（Sonnet 4使用時）
- **1リクエストあたり：約$0.044**

**Claude Code Max 20xの場合：**
- 月額$200で約900メッセージ/5時間
- 1日で理論上約4,500メッセージ可能
- **1メッセージあたり：約$0.044**（理論値）

**結論：**
単純なコストパフォーマンスはほぼ同等です。
**違いは制限の種類と柔軟性**にあります。

### 超過時のコスト比較

| 項目 | Cursor Ultra | Claude Code Max 20x |
|------|-------------|---------------------|
| **超過時の対応** | 追加料金で継続利用可 | 5時間待機が必要 |
| **追加料金** | API料金+20%マークアップ | 標準API料金（extra usage使用時） |
| **柔軟性** | 高い | 低い |

## 実際の使用パターン別の推奨

### パターン1：週末集中型

**特徴：**
- 平日はあまり使わない
- 週末に集中して開発する

**推奨：Cursor Ultra**
- 月次制限なので、週末に集中して使っても問題なし
- 月の前半に使い切っても、後半も使える

### パターン2：毎日コンスタント型

**特徴：**
- 毎日決まった時間に開発する
- 1日の使用量はそれほど多くない

**推奨：Claude Code Max 20x**
- 5時間ごとの制限で十分
- 毎日コンスタントに使うなら問題なし

### パターン3：プロジェクト締切型

**特徴：**
- 締切前に集中して開発する
- 使用量が予測しづらい

**推奨：Cursor Ultra**
- 月次制限なので柔軟に対応可能
- 超過時も追加料金で継続利用できる
