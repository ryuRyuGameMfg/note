# noteエディタ向け マークダウン安定運用ルール（実証ベース）

## 基本方針
- 引用ブロック（>）内はプレーンテキストのみ（装飾・リンク・太字禁止）
- 数字箇条書き（1. 2. 3.）は引用内で使用しない（囲み数字 ❶❷❸ を推奨）
- URLは独立行（前に説明文）

## 既知の不安定挙動と回避策

### 1) 引用内で「▼（三角）」の直後にリストや他要素を置くと崩れる
現象:
- `> ▼ 注意点` の直後に `- 箇条書き` や `1. 数字リスト`、` ``` `、`## 見出し` を置くとレンダリングが乱れることがある

回避:
- パターンA: 引用をいったん閉じ、通常文で箇条書きに切り替える
  
  例:
  
  > ▼ 注意点
  
  ・項目1
  ・項目2

- パターンB: 同一引用を維持するが、三角の次行を「テキスト行」でワンクッション入れてから箇条書きにする
  
  例:
  
  > ▼ 注意点
  > ｜詳細
  > ・項目1
  > ・項目2

- パターンC: 引用内の箇条書きをテキスト記号「・」で表現（`-`や`1.`は使わない）

### 2) 引用内の空行（`>`のみの行）が混入するとコピペ時に不具合
回避:
- 引用の空行は削除。段落区切りは通常行で表現

### 3) 引用とコードブロックの隣接で崩れることがある
回避:
- 引用直後にコードブロックを置かない。間に通常文1行を挟む
- 可能なら、コードは引用外に配置

### 4) 見出し（##/###）は引用の外に置く
- 引用内に見出しを生成しない

## チェックリスト（コピペ安定性）
- [ ] 引用内に太字/リンク/装飾なし
- [ ] 引用内に `1.` `-` を未使用（必要なら ❶/・/｜ を代替）
- [ ] `> ▼ ...` の直後に `-`/`1.`/```/## を配置していない
- [ ] 引用内の空行（`>`のみ）は存在しない
- [ ] 引用直後にコードブロックを置かない
