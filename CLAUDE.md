note記事・X投稿の制作ワークスペース。

## フォルダ構成

```
note/                    # note記事管理
├── 01_アイデア/
├── 02_下書き/
│   └── 02_執筆中/
├── 03_公開準備完了/
├── 04_公開済み/
└── 05_分析/

X/                       # X投稿管理
├── 01_アイデア/
├── 02_下書き/
├── 03_公開準備完了/
├── 04_公開済み/
└── 05_分析/

運用ルール/              # 執筆・運用ガイド
├── 01_執筆ガイド.md

├── 03_テンプレート.md
├── 04_AI運用.md
├── 05_X運用スタイル.md

```

## 記事ライフサイクル

```
アイデア → 執筆中 → 公開準備完了 → 公開 → 分析
```

**原則**: 1記事につき1ファイル。次の段階に進む際は古いファイルを削除して移動。

## クイックリファレンス

### note記事の基本ルール

- **文字数**: 2500-4000字
- **構成**: フック→問題→解決→結果→まとめ
- **見出し**: ##/###のみ（2階層まで）
- **引用ブロック**: 装飾なしのプレーンテキストのみ
- **URL**: 独立行に配置（前に説明文）
- **CTA**: 導入直後と末尾の2箇所

### X投稿の基本ルール

- **口調**: カジュアルで親しみやすい
- **バランス**: 宣伝2：日常8
- **投稿時間**: 平日20-22時 or 土曜10-12時
- **ハッシュタグ**: 2-4個

### CTA統一フォーマット（ココナラ版）

**導入直後（短縮版）**
```
**ゲーム開発のご相談：**
https://coconala.com/services/3327092
```

**末尾（フル版）** - 記事内容に応じて選択
- 技術記事 → パターンA
- AI関連 → パターンB
- 初心者向け → パターンC
- メタバース → パターンD

詳細は `運用ルール/06_CTAパターン.md` を参照
