## プロジェクト概要

note記事とX投稿の制作・管理・公開を効率化するワークスペース。

### 主な作業フロー
1. 記事執筆: アイデア → リサーチ → 下書き → 公開準備
2. リリース作業: NotePublisher（http://localhost:4003）でサムネイル生成・コピー・公開管理
3. X投稿: note記事の告知・独立投稿の作成

## 記事ライフサイクル

```
[1] アイデア発想
    |
[2] note/記事アイデア一覧.md に追記
    |
[3] note/01_クリップ/ でメモ・リサーチ・構成案作成
    |
[4] note/02_下書き/ で執筆（クリップは削除）
    |
[5] note/03_公開準備完了/ で最終チェック（下書きは削除）
    |
[6] NotePublisher でリリース作業
    ├── タイトル・本文コピー
    ├── サムネイル・インフォグラフダウンロード
    └── 公開済みチェック → note/04_公開済み/ に移動
    |
[7] 分析 → note/05_分析/ で効果測定
```

原則: 1記事につき1ファイル。次の段階に進む際は古いファイルを削除して移動。

## NotePublisher（リリースツール）

### ポート番号レギュレーション
固定ポート: 4003 - 変更禁止

### 起動方法
```bash
cd NotePublisher
npm run dev
# http://localhost:4003 で起動
```

### 主な機能
- 記事一覧表示（ダークモード）
- タイトル・本文のワンクリックコピー
- サムネイル・インフォグラフのダウンロード
- ダーク/ホワイトモード切り替え（インフォグラフ）
- 公開管理（公開済みチェック・フォルダ移動）

詳細は `NotePublisher/README.md` を参照。

## SEO・アルゴリズム対応（重要）

### note公式アルゴリズム
| 対応事項 | 理由 |
|---------|------|
| 冒頭3行に要約 | meta descriptionとして自動使用 |
| ハッシュタグ3-5個 | note公式推奨 |
| Xで告知→スキ10個 | アルゴリズム起動 |

### X公式アルゴリズム
| 対応事項 | 理由 |
|---------|------|
| 質問で締める | 返信=いいねの27倍 |
| 外部リンクは返信欄 | 本文に入れると低評価 |
| 返信には必ず返信 | 返信への返信=150倍 |
| 画像添付 | 1.7倍拡散効果 |

詳細は `DOCS/04_SEO/公式アルゴリズム情報.md` を参照。
