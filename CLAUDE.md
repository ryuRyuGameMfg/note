# プロジェクト概要

note記事・X投稿の制作ワークスペース。

## フォルダ構成

```
note/（note記事管理）
├── 記事アイデア一覧.md（全記事アイデア・プロジェクト計画）
├── 01_クリップ/（記事メモ・リサーチ・作業場）
├── 02_下書き/（執筆中の記事）
├── 03_公開準備完了/（最終チェック待ち）
├── 04_公開済み/
└── 05_分析/

X/（X投稿管理）
├── 01_アイデア/
├── 02_下書き/
├── 03_公開準備完了/
├── 04_公開済み/
└── 05_分析/

NotePublisher/（noteリリースツール - 固定ポート: 4003）
├── README.md（使用方法・技術仕様）
├── REGULATIONS.md（デザインレギュレーション）
└── src/（Next.jsアプリケーション）
    ├── app/（ページ・API）
    ├── components/（UIコンポーネント）
    └── lib/（ユーティリティ）

export/（エクスポート用）
```

## AIへの指示

記事作成: `/NoteWrite` または `/NoteRewrite` スキルを使用
X投稿作成: `/XWrite` スキルを使用

## 記事ライフサイクル（note）

```
アイデア発想
    ↓
note/記事アイデア一覧.md に追記
    ↓
note/01_クリップ/ でメモ・リサーチ・構成案作成
    ↓
note/02_下書き/ で執筆（クリップのファイルは削除）
    ↓
note/03_公開準備完了/ で最終チェック（下書きのファイルは削除）
    ↓
NotePublisher でリリース作業（http://localhost:4003）
    ├── タイトル・本文コピー
    ├── サムネイル・インフォグラフダウンロード
    └── 公開済みチェック → note/04_公開済み/ に移動
    ↓
分析 → note/05_分析/ で効果測定
```

**原則**: 1記事につき1ファイル。次の段階に進む際は古いファイルを削除して移動。

## NotePublisher（noteリリースツール）

公開準備完了の記事を効率的にリリースするためのツールです。

### ポート番号レギュレーション
**固定ポート: 4003** - 他のローカルサービスとの競合を避けるため、このポート番号は変更禁止です。

### 主な機能
- 記事一覧表示（ダークモード）
- タイトル・本文のワンクリックコピー
- サムネイル・インフォグラフのダウンロード
- ダーク/ホワイトモード切り替え（インフォグラフ）
- 公開管理（公開済みチェック・フォルダ移動）

### 使用方法
```bash
cd NotePublisher
npm run dev
# http://localhost:3010 で起動
```

詳細は `NotePublisher/README.md` を参照してください。
