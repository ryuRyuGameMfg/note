# note運用方針

最終更新: 2025年12月

## コンセプト

ミッション: ゲーム開発・AI技術の実践知を、わかりやすく共有する

ポジショニング:
- 誰が: 235件以上の実績を持つ24歳の個人開発者
- 何を: Unity開発・AI技術の実践ノウハウ
- 誰に: ゲーム開発に興味がある初心者〜中級者
- どう: 失敗談も含めた正直なストーリーで

差別化ポイント:

| 競合 | りゅうやの強み |
|------|---------------|
| 大手メディア | 個人の生の声、失敗談も含むリアルな体験 |
| 技術ブログ | 初心者にもわかりやすい言葉、具体的な数字 |
| インフルエンサー | 実績に裏付けられた実践的ノウハウ |

## ターゲット

メインターゲット:

| 項目 | 詳細 |
|------|------|
| 属性 | 20-35歳、男性中心、エンジニア or エンジニア志望 |
| 関心 | Unity、ゲーム開発、副業、AI技術 |
| 課題 | 開発で行き詰まっている、収益化したい、効率化したい |
| 行動 | 技術記事を検索、Xで情報収集、ココナラで外注検討 |

ペルソナ:
- 名前: たける（27歳）
- 職業: SIerで働くエンジニア（3年目）
- 副業: Unityでゲーム開発を始めたい
- 課題: 何から始めればいいかわからない、時間がない
- 情報源: note、X、YouTube、Qiita

刺さるコンテンツ:
- 具体的な数字（月収70万、235件、15分で完了）
- 失敗談→克服のストーリー
- 実際に使えるテクニック・ツール
- 初心者でもできる〇〇

刺さらないコンテンツ:
- 抽象的な成功論
- 専門用語だらけの解説
- 上から目線のアドバイス
- 宣伝色が強い記事

## KPI・目標

短期目標（3ヶ月）:

| 指標 | 目標値 |
|------|--------|
| 月間PV | 5,000 |
| フォロワー | 500 |
| 記事数 | 12本（週1本） |
| ココナラ流入 | 月10件 |

中期目標（1年）:

| 指標 | 目標値 |
|------|--------|
| 月間PV | 20,000 |
| フォロワー | 2,000 |
| 記事数 | 50本 |
| 有料記事売上 | 月5万円 |

重要指標の優先順位:
1. ココナラへの流入数（収益直結）
2. スキ数（記事の質・共感度）
3. PV数（認知・SEO効果）
4. フォロワー数（継続的なリーチ）

## コンテンツ戦略

記事カテゴリ:

| カテゴリ | 割合 | 目的 | CTA |
|---------|------|------|-----|
| 技術ノウハウ | 40% | SEO流入、専門性 | Unity開発相談 |
| 実績・体験談 | 30% | 信頼性、共感 | メイン受託開発 |
| AI関連 | 20% | 差別化、新規層 | AIキャラクター |
| 初心者向け | 10% | 入口、ファン獲得 | 画面共有相談 |

記事の役割:
1. 認知: SEO記事で検索から流入
2. 興味: ストーリー記事で人となりを知る
3. 信頼: 実績記事でこの人なら任せられると思わせる
4. 行動: CTAでココナラに誘導

更新頻度:

| 項目 | 目標 |
|------|------|
| 新規記事 | 週1本 |
| 既存記事リライト | 月2本 |
| X告知 | 記事公開時 + 週2-3回 |

## SEO戦略

狙うキーワード:

| 優先度 | キーワード例 | 検索意図 |
|--------|-------------|---------|
| 高 | Unity 初心者 始め方 | 入門者獲得 |
| 高 | Unity 効率化 時短 | 中級者獲得 |
| 中 | AI アバター 接客 | AIサービス訴求 |
| 中 | ゲーム開発 副業 | 副業層獲得 |
| 低 | Cluster ワールド 作り方 | メタバース層 |

SEOとCTAの連携:

| 記事テーマ | 流入KW | 最適CTA |
|-----------|--------|---------|
| Unity入門 | Unity 初心者 | 画面共有相談 |
| Unity時短 | Unity 効率化 | Unity開発相談 |
| AI活用 | AI 接客 導入 | AIキャラクター |
| 実績系 | - | メイン受託開発 |

## 収益戦略

収益源の優先順位:
1. ココナラ受託（メイン収益）: note → 信頼構築 → ココナラ流入
2. 有料記事（サブ収益）: ノウハウ系の深掘り版
3. アフィリエイト（補助）: 開発ツール・書籍紹介

CTAの使い分け:

| 記事タイプ | CTA優先度 |
|-----------|----------|
| 技術記事 | Unity相談 > メイン受託 |
| AI記事 | AIキャラクター > メイン受託 |
| 初心者向け | 画面共有相談 > チャット相談 |
| 実績系 | メイン受託 > AIキャラクター |

## 品質基準

必須要件:
- 2500-4000字
- 具体的な数字・実績を含む
- デメリット・注意点を正直に記載
- 冒頭3行に要約（meta description対応）
- CTA2箇所（導入後・末尾）

差別化要素（最低1つ）:
- 実体験に基づくストーリー
- 具体的なBefore/After（数字）
- 他の記事にない独自の視点
- 失敗談と克服方法

## 運用フロー

記事作成フロー:
1. アイデア出し: ターゲットの課題、どのCTAに繋げるか
2. 構成作成: 04_SEO/公式アルゴリズム情報.md参照、冒頭3行を先に書く
3. 執筆: 02_note/テンプレート.md使用、人間味のある文体で
4. チェック: 02_note/チェックリスト.mdで確認
5. 公開: Xで告知（スキ10個目標）
6. 分析: 1週間後にPV/スキ確認、良かった点/改善点をメモ

月次振り返り:
- 月間PV確認
- 人気記事の傾向分析
- ココナラ流入数確認
- 来月のコンテンツ計画
