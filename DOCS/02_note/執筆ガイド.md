# note執筆ガイド

参照: 04_SEO/公式アルゴリズム情報.md

## 記事ライフサイクル

アイデア → 執筆中 → 公開準備完了 → 公開 → Xで告知 → 分析 → 改善

原則:
- 1記事につき1ファイルのみ保持
- 次の段階に進む際は古いファイルを削除して移動
- 公開直後のXでの告知は必須（スキ10個でアルゴリズム起動）

## 公式アルゴリズム対応

| 対応事項 | 理由 | 優先度 |
|---------|------|--------|
| 冒頭3行に要約を入れる | meta descriptionとして自動使用される | 必須 |
| 自己紹介・広告を冒頭に入れない | 検索結果で「こんにちは」が表示されてしまう | 必須 |
| 公開後すぐにXで告知 | スキ10個以上でアルゴリズムが動き出す | 必須 |
| ハッシュタグは3〜5個 | note公式推奨 | 推奨 |
| 見出し画像は1280×670px | note公式推奨サイズ | 推奨 |

## ファクトチェック

執筆前・執筆中の必須事項:
- 記載する数字・データは必ず最新の公式ソースで確認する
- API料金、製品価格、統計データなどは執筆時点の最新情報を使用
- 引用する研究論文や記事は実在するものか確認
- 体験談以外の情報は客観的なソースに基づく
- 不確実な情報は「〜と言われている」「〜の可能性がある」など表現を調整

## 冒頭の書き方

noteの冒頭部分はmeta descriptionとして自動使用される。検索結果に表示されるためクリック率に直結。

冒頭3行の構成:
- 1行目: 読者の悩み or 記事の結論
- 2行目: この記事で得られること
- 3行目: 権威性（実績の数字）

良い例:
```
「Unityで効率化したいけど、何から始めればいいかわからない」

この記事では、235件の開発実績から見つけた
15分でできる時短テクニックを公開します。
```

悪い例:
```
こんにちは、りゅうやです。
今日は開発についてお話しします。
よろしくお願いします。
```

## タイトルの書き方

数字 + 具体的な成果 + ターゲット明確化

効果的な要素:
- 具体的な数字（期間、金額、倍率）
- 明確な成果やベネフィット
- 想定読者が自分事と感じる表現
- キーワードは前半に配置（SEO重要）

避けるべきパターン:
- 抽象的で曖昧な表現
- 読者メリットが不明確
- ターゲットが広すぎる
- 長すぎるタイトル（省略される）

## 基本構成（2500-4000文字）

1. 冒頭3行（200字）: 要約（meta description用）
2. CTA（導入直後）
3. 問題提起（300字）: なぜ重要か
4. 解決策（1500字）: 具体的な方法
5. 実践結果（800字）: Before/After（数字で）
6. デメリット・注意点（200字）: 信頼性UP
7. まとめ（200字）: 行動喚起 + 質問
8. CTA（末尾）

## ストーリー型記事の構成

1. フック（衝撃的な一文）
2. 過去の状況（共感を得る）
3. きっかけ（変化の始まり）
4. 失敗・葛藤（リアリティ）
5. 転機（発見・気づき）
6. 成功体験（具体的な効果・数字）
7. デメリットも正直に（信頼性）
8. 読者へのメッセージ（行動喚起 + 質問）

## 文体

避けるべき表現:
- 「以下の通りです」
- 「〜することができます」
- 「効果的です」

代わりに使う表現:
- 「こんな感じ」
- 「〜できる」
- 「めっちゃいい」

## 装飾ルール

見出し:
- 2階層まで（## と ### のみ）
- 見出しにキーワードを含める（SEO効果）

太字:
- 1段落に1〜2個まで

引用ブロック:
- 引用ブロック内は装飾なしのプレーンテキストのみ
- 引用内に太字、斜体、数字箇条書きを使用しない
- 空の引用行（> のみ）を作らない

URL配置:
- 必ず独立した行に配置

水平線:
- 原則として使用しない

チェック項目・チェックリスト:
- 記事内に「あなたはこういうのができてますか？」のようなチェック項目・セクションを書かない
- チェックリストは記事の質を弱め、読みにくくし、浅い記事になってしまう
- 実際にチェックできないため読者にとって邪魔になる
- 判断基準やルールは説明文として自然に組み込む

## CTA配置

配置位置:
1. 導入直後（冒頭3行の後、本編開始前）
2. 記事末尾（まとめの後）

注意点:
- CTAの後に追伸は書かない
- 押し付けがましくしない
- 記事内容と関連性を保つ

## 公開後フロー

1. 公開
2. すぐにXで告知（リンクは返信欄）
3. 告知投稿に自分で返信（URLを投稿）
4. スキ10個を目指す
5. コメント・返信に対応
6. 1週間後に分析（PV/スキ数）
