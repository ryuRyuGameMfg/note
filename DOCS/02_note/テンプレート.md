## 冒頭の書き方

noteの冒頭部分はmeta descriptionとして自動使用される。自己紹介・広告ではなく記事の要約を入れる。

冒頭3行の構成:
- 1行目: 読者の悩み or 記事の結論
- 2行目: この記事で得られること
- 3行目: 権威性（実績の数字）

良い例:
```
「Unityで効率化したいけど、何から始めればいいかわからない」

この記事では、235件の開発実績から見つけた
15分でできる時短テクニックを公開します。
```

悪い例:
```
こんにちは、りゅうやです。
今日は開発についてお話しします。
よろしくお願いします。
```

## テンプレート使用時の注意

| 項目 | 対応 |
|------|-----|

| 公開直後 | Xで告知してスキ10個を目指す |
