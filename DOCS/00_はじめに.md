# 運用ルール - はじめに

AIが記事・投稿を作成する際に最初に読むファイル。

## 読む順番

### note記事を作成する場合

1. 01_運用方針/note運用方針.md（コンセプト、ターゲット、KPI、戦略）
2. 04_SEO/公式アルゴリズム情報.md（冒頭3行、ハッシュタグなど公式対応）
3. 02_note/テンプレート.md（記事の型）
4. 02_note/チェックリスト.md（公開前確認）
5. 出力

### X投稿を作成する場合

1. 01_運用方針/X運用方針.md（コンセプト、ターゲット、KPI、戦略）
2. 04_SEO/公式アルゴリズム情報.md（質問で締める、リンクは返信欄など公式対応）
3. 03_X/テンプレート.md（投稿の型）
4. 03_X/チェックリスト.md（公開前確認）
5. 出力

### note記事をXで告知する場合

1. 04_SEO/公式アルゴリズム情報.md（X側のアルゴリズム対応）
2. 03_X/テンプレート.md（note記事告知型を使用）
3. 03_X/チェックリスト.md（note記事告知時の追加チェック）
4. 出力（本文 + 返信欄用URL）

## フォルダ構成

```
note/（note記事管理）
├── 📋 記事アイデア一覧.md（全記事アイデア・プロジェクト計画）
├── 01_クリップ/（記事メモ・リサーチ・作業場）
├── 02_下書き/（執筆中の記事）
├── 03_公開準備完了/（最終チェック待ち）
├── 04_公開済み/
└── 05_分析/

X/（X投稿管理）
├── 01_アイデア/
├── 02_下書き/
├── 03_公開準備完了/
├── 04_公開済み/
└── 05_分析/

運用ルール/
├── 00_はじめに.md（このファイル）
├── 01_運用方針/（コンセプト・戦略）
│   ├── note運用方針.md
│   └── X運用方針.md
├── 02_note/（note記事作成時）
│   ├── 執筆ガイド.md
│   ├── チェックリスト.md
│   └── テンプレート.md
├── 03_X/（X投稿作成時）
│   ├── 執筆ガイド.md
│   ├── チェックリスト.md
│   └── テンプレート.md
├── 04_SEO/（公式アルゴリズム）
│   ├── 公式アルゴリズム情報.md（最重要）
│   └── 一般テクニック集.md
└── 05_共通/
    ├── CTAパターン.md
    └── AI運用.md
```

## 記事ライフサイクル（note）

```
💡 アイデア発想
    ↓
📋 note/記事アイデア一覧.md に追記
    ↓
📝 note/01_クリップ/ でメモ・リサーチ・構成案作成
    ↓
✍️ note/02_下書き/ で執筆（クリップのファイルは削除）
    ↓
✅ note/03_公開準備完了/ で最終チェック（下書きのファイルは削除）
    ↓
🚀 公開 → note/04_公開済み/ に移動
    ↓
📊 分析 → note/05_分析/ で効果測定
```

**重要**: 各段階に進む際は、前の段階のファイルを削除してから移動する。1記事につき1ファイルを原則とする。

## 最重要ポイント

### note記事

| 項目 | 対応 |
|------|------|
| 冒頭3行 | 記事の要約を入れる（自己紹介・広告NG） |
| ハッシュタグ | 3〜5個 |
| 公開後 | Xで告知してスキ10個を目指す |

### X投稿

| 項目 | 対応 |
|------|------|
| 質問で締める | 返信=いいねの27倍の価値 |
| 外部リンク | 本文NG、返信欄に配置 |
| 返信対応 | 来た返信には必ず返信（150倍の価値） |

## 補足

詳細が必要な場合：
- 執筆の書き方詳細：02_note/執筆ガイド.md or 03_X/執筆ガイド.md
- CTA選択：05_共通/CTAパターン.md
- AI運用ルール：05_共通/AI運用.md
- 一般的なSEOテクニック：04_SEO/一般テクニック集.md

優先度：
1. 最優先：04_SEO/公式アルゴリズム情報.md
2. 重要：01_運用方針/
3. 参照：02_note/ or 03_X/
4. 補足：05_共通/
