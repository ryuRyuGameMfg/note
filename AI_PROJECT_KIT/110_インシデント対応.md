# インシデント対応（事故時テンプレ）

対象：秘密情報の混入、破壊的変更、誤実装の大量混入、外部送信など。

## 0) まずやる（5分）

- 影響範囲の特定（どの環境/どのユーザー/どのデータ）
- 被害拡大の停止（マージ停止、ロールバック、権限停止）
- 証跡確保（ログ、PR差分、実行コマンド履歴）

## 1) 状況整理テンプレ

```text
### 何が起きた？
- 

### いつ/どこで？
- 

### 影響範囲
- 

### 直近の変更（PR/コミット）
- 

### 一時対応（止血）
- 

### 恒久対応（再発防止）
- 

### 学び（ルール/テンプレ更新）
- 更新先：10 / 60 / 50 / 100 / 90
- 更新内容：
```

## 2) AI利用時の再発防止パターン（例）

- 指示が曖昧だった → `30_タスク依頼テンプレ.md` に必須欄を増やす
- テストが弱かった → `50_チェックリスト.md` に境界条件の観点を追加
- 破壊的操作が走った → `60_禁止事項と安全運用.md` を強化、権限を絞る
- 仕様を推測した → `10_AI指示書_プロジェクト雛形.md` に「不明点は実装前に列挙」を追加
