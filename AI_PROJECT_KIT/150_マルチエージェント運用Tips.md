# マルチエージェント運用Tips（2025最新）

目的：1つのAIだけでなく、**複数の専門エージェントを組み合わせる**とさらに効率が上がる。

## 1. マルチエージェントとは

- **役割ごとに専門化したAIを用意**し、並列/直列で協調させる
- 例：
  - 設計エージェント（Plan特化）
  - 実装エージェント（Act特化）
  - レビューエージェント（Reflect/品質特化）
  - ドキュメントエージェント（ドキュメント生成特化）

## 2. Cursorでの簡易実現

Cursor自体は単一エージェントだが、**モードの切り替え**で擬似的に役割分担できる。

- **Askモード**：調査・設計・比較（Planフェーズ）
- **Agentモード**：実装・テスト（Actフェーズ）
- **終了後Askモード**：セルフレビュー・振り返り（Reflectフェーズ）

## 3. 本格的なマルチエージェント（外部ツール併用）

- **Claude Code + Cursor**：仕様検討はClaude、実装はCursor
- **GitHub Copilot Coding Agent**：PR単位でタスクを自動実行
- **MCPサーバー**：外部ツール連携（DB、API、監視）を自動化

## 4. オーケストレーションのコツ

- **責務を明確に分ける**：1エージェント=1責務
- **受け渡しの形式を固定**：JSON/Markdown/Task Briefで統一
- **失敗時のフォールバック**：エラー時は人間にエスカレーション
- **ログを残す**：どのエージェントが何をしたかをトレース可能に

## 5. 注意点

- エージェントが増えると"伝言ゲーム"でズレやすい
- 最初は「1エージェント + モード切り替え」で十分
- 複雑化したら徐々に分離
