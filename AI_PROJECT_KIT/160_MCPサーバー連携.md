# MCPサーバー連携（2025最新）

目的：Model Context Protocol（MCP）を使って、**AIに外部ツール/データソースを接続**する。

## 1. MCPとは

- Anthropic主導で標準化された「AIと外部ツールを繋ぐプロトコル」
- AIが自律的にDB/API/ファイルシステム等にアクセスできる
- 2025年は非同期タスク、セキュリティ強化、Streamable HTTPが標準化

## 2. 何ができる？

- **DBアクセス**：SQLを発行してデータ取得
- **API連携**：外部サービスを呼び出し
- **ファイル操作**：ドキュメント読み書き
- **監視/ログ**：メトリクス取得、アラート
- **CI/CD連携**：テスト実行、デプロイ状況確認

## 3. Cursorでの利用

- `Cursor Settings` → `MCP` → サーバーを追加
- よく使うMCPサーバー例（※パッケージ名は変更される可能性があるため、公式で確認推奨）：
  - `@modelcontextprotocol/server-filesystem`：ファイル操作
  - `@modelcontextprotocol/server-github`：GitHub連携
  - `@modelcontextprotocol/server-postgres` / `server-sqlite`：DB連携
  - `@anthropic/claude-code`：Claude Code内蔵ツール

参考：[MCP Servers一覧（公式）](https://github.com/modelcontextprotocol/servers)

## 4. 運用上の注意

- **認可をかける**：MCPサーバーに渡す権限は最小限
- **破壊的操作は禁止 or 承認制**：DELETE/DROP等
- **ログを残す**：何を実行したかトレース可能に
- **機密データの扱い**：MCPサーバー経由でも秘密情報はマスク

## 5. 簡単な導入ステップ

1. `npx @anthropic/create-mcp-server` で雛形作成
2. 必要なツールを実装（関数定義）
3. CursorのMCP設定に追加
4. 動作確認（Agentモードで呼び出す）
