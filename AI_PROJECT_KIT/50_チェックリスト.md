# チェックリスト（DoD / レビュー / セキュリティ）

## DoD（Definition of Done）

- [ ] ACを満たす
- [ ] 回帰がない（スモーク/回帰テスト）
- [ ] lint/typecheck/test が通る
- [ ] エラーハンドリング・境界条件がある
- [ ] 必要なドキュメントが更新されている

## レビュー観点（LLM生成コードで落ちやすい）

- [ ] 暗黙前提が増えていない（入力/状態/順序）
- [ ] エッジケース（空/null/0件/巨大入力/権限なし）
- [ ] セキュリティ（入力検証、認可、秘密情報、依存関係）
- [ ] パフォーマンス（無駄なIO/ループ/N+1）
- [ ] 可読性（命名/責務/重複）

## セキュリティ最低限

- [ ] 機密情報がログ/レスポンス/例外に混入していない
- [ ] 入力検証がある（サーバ側）
- [ ] 認可が必要な箇所に抜けがない
- [ ] 依存関係追加の理由が説明されている
