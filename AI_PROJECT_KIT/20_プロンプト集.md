# プロンプト集（汎用）

このファイルは、AIに毎回同じ品質で動いてもらうための“定型句”です。


## 0. 毎回先頭に付ける安全プリフィックス

```text
重要ルール：
- 変更は小さく分割（1PR=1目的）
- 破壊的操作は実行しない（提案のみ）
- 秘密情報は出力しない/要求しない
- 実装後は必ず lint/typecheck/test を通す
- 既存の命名・設計・ディレクトリ規約に合わせる
```

## 1. 調査（Ask向け）

### 影響範囲調査

```text
目的：<目的>
質問：この変更の影響範囲（ファイル/関数/コンポーネント）を列挙し、理由も書いて。
出力：
- 影響箇所
- リスク
- 最小実装プラン（3〜6ステップ）
```

### 実装案比較（2〜3案）

```text
<目的> を実現する案を3つ。
各案：変更点/メリット/デメリット/リスク/テスト方針。
最後に推奨案と理由。
制約：<制約>
```

## 2. 実装（Agent向け）

### Task Briefを満たして実装

```text
以下のTask Briefを満たすように実装して。

<Task Brief>

進め方：
- 変更計画を短く提示
- 実装
- lint/typecheck/test を実行して通す
- 失敗したら原因→修正→再実行
- PR用要約を出す
```

## 3. テスト

### ACからテスト設計→実装

```text
ACからテストケースを列挙して（正常/異常/境界/権限）。
優先度の高い順に実装して。
AC：
- [ ] ...
```

## 4. レビュー（セルフレビュー）

### PR差分レビュー

```text
この差分をレビューして。
観点：仕様/エッジケース/セキュリティ/性能/可読性。
重要度 high/medium/low で指摘し、修正案も。
```

## 5. ドキュメント更新

### 変更に合わせてdoc更新

```text
この変更に合わせて更新すべきドキュメントを提案し、更新案も作って。
ユーザー向け/開発者向けに分けて。
```
