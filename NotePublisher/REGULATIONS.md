# サムネイル・インフォグラフ作成レギュレーション

**最終更新**: 2025年12月18日

このドキュメントは、サムネイルとインフォグラフを作成する際の**必須ルール**です。
**全てのコンテンツはこのレギュレーションに従って作成してください。**

---

## 📐 サイズ・解像度

### サムネイル
- **サイズ**: 1280×670px（note公式推奨）
- **フレーム**: 画像がはみ出さないよう、内側に余白（padding: 48px）
- **プレビュー**: 50%スケール表示（640×335px）

### インフォグラフ
- **幅**: 800px固定
- **高さ**: 可変（600px〜）
- **フレーム**: 画像がはみ出さないよう、内側に余白（padding: 48px）
- **プレビュー**: 50%スケール表示（400px〜）

## 🎨 カラーパレット定義

### 1. ブルー系（メインカラー）
**用途**: アプリケーションUI、アクセント、グラデーション

```
Blue-950: #172554  // 最深部背景
Blue-900: #1e3a8a  // 深い背景
Blue-800: #1e40af  // カード背景
Blue-700: #1d4ed8  // ボーダー濃
Blue-600: #2563eb  // プライマリボタン
Blue-500: #3b82f6  // アクセント
Blue-400: #60a5fa  // ハイライト
Blue-300: #93c5fd  // テキスト明
Blue-200: #bfdbfe  // 背景薄
```

**グラデーション例**:
```css
/* 背景 */
background: linear-gradient(135deg, #172554 0%, #1e3a8a 50%, #172554 100%);

/* カード */
background: linear-gradient(135deg, #1e3a8a 0%, #1e40af 100%);

/* ボタン */
background: linear-gradient(90deg, #2563eb 0%, #3b82f6 100%);
```

### 2. グレー系（補助カラー）
**用途**: テキスト、ボーダー、背景

```
Gray-950: #0a0a0a  // 最深部
Gray-900: #171717  // ダーク背景
Gray-800: #262626  // カード背景
Gray-700: #404040  // ボーダー
Gray-600: #525252  // テキスト薄
Gray-400: #a3a3a3  // プレースホルダー
Gray-300: #d4d4d4  // テキスト明
Gray-200: #e5e5e5  // 薄い背景
White:    #ffffff  // メインテキスト
```

### 3. アクセントカラー（用途別）
**成功/肯定**: Green
```
Green-500: #22c55e  // チェックマーク、メリット
Green-400: #4ade80  // ハイライト
```

**エラー/否定**: Red
```
Red-500:   #ef4444  // エラー、デメリット
Red-400:   #f87171  // ハイライト
```

**警告**: Yellow/Orange
```
Yellow-500: #eab308  // 警告
Orange-500: #f97316  // 注目
```

**情報**: Cyan/Purple
```
Cyan-400:   #22d3ee  // 情報
Cyan-500:   #06b6d4  
Purple-500: #a855f7  // 特別
Purple-400: #c084fc
```

### 4. グラデーション組み合わせ
```css
/* サムネイル用 */
.gradient-dark {
  background: linear-gradient(135deg, #0a0a0a 0%, #1e3a8a 50%, #0a0a0a 100%);
}

/* インフォグラフ用（ダークモード） */
.infograph-dark {
  background: #171717;
  color: #ffffff;
}

/* インフォグラフ用（ホワイトモード） */
.infograph-light {
  background: #ffffff;
  color: #171717;
}
```

## 🎯 デザインルール

### アイコン使用規則
- ✅ **SVGアイコンのみ使用**（lucide-react推奨）
- ❌ **絵文字完全禁止**（サムネイル・インフォグラフ共に）
- **サイズ**: 24px〜32px（インフォグラフ）、48px〜64px（サムネイル）
- **カラー**: カラーパレットから選択
- **ストローク**: strokeWidth={2}〜{3}

### 推奨SVGアイコン（lucide-react）
```tsx
// 肯定・成功
<Check className="w-6 h-6 text-green-500" strokeWidth={3} />
<TrendingUp className="w-6 h-6 text-green-500" />

// 否定・エラー
<X className="w-6 h-6 text-red-500" strokeWidth={3} />
<TrendingDown className="w-6 h-6 text-red-500" />

// 情報
<Info className="w-6 h-6 text-blue-400" />
<FileText className="w-6 h-6 text-blue-400" />
<Calendar className="w-6 h-6 text-blue-400" />

// 機能
<Download className="w-6 h-6 text-blue-500" />
<Copy className="w-6 h-6 text-blue-500" />
<ArrowRight className="w-6 h-6 text-blue-400" />
```

### 装飾ルール
- ✅ グラデーション背景（ブルー系）
- ✅ グラスモーフィズム効果（backdrop-blur）
- ✅ シンプルで洗練されたデザイン
- ✅ 適度なシャドウ（shadow-lg, shadow-xl）
- ❌ 過度な装飾は避ける

## ✍️ テキストルール

### タイトル
- **文字数**: 20〜35文字
- **改行**: 【】の後に改行を入れる
- **構成**: 数字 + 具体的な成果 + ターゲット明確化

**良い例**:
```
【2025年12月最新】
Cursor Ultra vs Claude Code Max 20x｜コストパフォーマンス徹底比較
```

### マークダウン表記
- **禁止**: アスタリスク（*）、アンダースコア（_）をそのまま表示
- **対応**: HTMLタグまたはReactコンポーネントでレンダリング

## 📊 インフォグラフ特有ルール

### 統一レイアウト規則
**全インフォグラフで厳守**:
- **幅**: 800px固定 (`w-[800px]`)
- **最小高さ**: 600px (`min-h-[600px]`)
- **パディング**: 48px (`p-12`)
- **余白**: コンテンツに`flex flex-col`を使用し、`flex-1`で下部余白を自動調整
- **タイトルサイズ**: 3xl (`text-3xl`)
- **本文サイズ**: base (`text-base`)
- **アイコンサイズ**: 24px (`w-6 h-6`)

### マークダウン処理
- **必須**: 全テキストに `parseMarkdownText()` を適用
- **太字**: `**テキスト**` → `<strong className="font-bold">テキスト</strong>`
- **アスタリスク露出禁止**: 生のマークダウン記法を表示しない

### カラー使用規則

#### 比較表（ComparisonTable）
```tsx
// ヘッダー
ダーク: bg-gray-800, text-white, border-gray-700
ライト: bg-gray-100, text-gray-800, border-gray-200

// チェックマーク
成功: text-green-500 (strokeWidth={3})
失敗: text-red-500 (strokeWidth={3})
```

#### メリット・デメリット（ProsCons）
```tsx
// メリット
ダーク: bg-green-900/20, border-green-500/30, text-green-400
ライト: bg-green-50, border-green-200, text-green-700
アイコン: <TrendingUp className="w-6 h-6" />

// デメリット
ダーク: bg-red-900/20, border-red-500/30, text-red-400
ライト: bg-red-50, border-red-200, text-red-700
アイコン: <TrendingDown className="w-6 h-6" />
```

#### 統計データ（StatsInfographic）
```tsx
// カラーローテーション（順番に使用）
const colors = [
  '#3b82f6', // Blue-500
  '#22c55e', // Green-500
  '#f97316', // Orange-500
  '#a855f7', // Purple-500
  '#22d3ee', // Cyan-400
  '#2563eb'  // Blue-600
];

// 数値表示
text-4xl font-black (カラーはcolorsから)
```

#### バーチャート（BarChart）
```tsx
// バー
ダーク: bg-gray-800 (背景), バーはcolorsから
ライト: bg-gray-200 (背景), バーはcolorsから

// 高さ: h-8 (32px)
// 角丸: rounded-full
```

### 余白の厳格な管理
```tsx
// 正しい例（余白が自然）
<div className="w-[800px] min-h-[600px] p-12 flex flex-col">
  <h2 className="text-3xl font-bold mb-6">タイトル</h2>
  <div className="flex-1"> {/* ← ここがポイント */}
    {/* コンテンツ */}
  </div>
</div>

// 間違った例（下部に不自然な余白）
<div className="w-[800px] min-h-[600px] p-12">
  <h2 className="text-3xl font-bold mb-6">タイトル</h2>
  <div>
    {/* コンテンツ */}
  </div>
  {/* ← ここに無駄な余白ができる */}
</div>
```

### 比較表（ComparisonTable）
- **ヘッダー**: 太字＋背景色（ダーク: bg-gray-800, ライト: bg-gray-100）
- **罫線**: 最小限（border-gray-700 / border-gray-200）
- **最終行**: 下ボーダーなし（不自然な余白を防ぐ）
- **パディング**: p-4（統一）
- **チェックマーク**: `<Check>` / `<X>` SVGアイコン
- **カラー**: Green-500（✓）、Red-500（✗）

### メリット・デメリット（ProsCons）
- **レイアウト**: `flex gap-6` で左右均等
- **アイコン**: `<TrendingUp>` / `<TrendingDown>`
- **カラー**: Green-400（メリット）、Red-400（デメリット）
- **背景**: 半透明の色付き背景（bg-green-900/20 など）
- **リスト間隔**: `space-y-3`
- **`flex flex-col`**: 両側のカードに適用し、`flex-1`でリスト部分を伸ばす

### 統計データ（StatsInfographic）
- **グリッド**: `grid-cols-3 gap-6`
- **数値**: 大きく目立つフォント（text-4xl font-black）
- **単位**: 数値に含める
- **カラー**: カラーパレットから順番に使用
- **ボーダー**: トップのみ 4px solid

### バーチャート（BarChart）
- **バー**: 高さ h-8（32px）、rounded-full
- **カラー**: カラーパレットから順番に使用
- **レイアウト**: `space-y-5` + `flex flex-col justify-center`
- **ラベル幅**: w-32（統一）
- **数値幅**: w-16（統一）

## 🚫 禁止事項

### 絶対に守ること
1. ❌ **絵文字の使用禁止**（サムネイル・インフォグラフ共に）
2. ❌ **マークダウン表記の露出**（*, _, #など）
3. ❌ **不自然な改行**（装飾文字の途中で改行）
4. ❌ **フレームからのはみ出し**
5. ❌ **読めないほど小さいフォント**（14px以下）
6. ❌ **カラーパレット外の色**
7. ❌ **情報の詰め込みすぎ**

### 推奨事項
- ✅ SVGアイコンのみ使用
- ✅ カラーパレットから色を選択
- ✅ 適度な余白を確保
- ✅ 階層を明確に
- ✅ コントラストを意識
